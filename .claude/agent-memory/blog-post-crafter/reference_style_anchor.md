Style anchor: `/Users/rafael/Masters/remg1997.github.io/redesign/src/content/posts/why-this-blog.md`.

Read it before drafting any new post. It establishes:
- The first-person, conversational voice.
- H2/H3 structure with scannable headers.
- Use of short bullet lists for enumerating claims.
- Links to prior work inline, sparingly, for credibility.
- The "Two tracks, one mission" framing — posts should feel consistent with either the "For Engineers" or "For Everyone" positioning.
- Ending with a call to dialogue rather than a summary.

When voice drifts, come back to this file.
